# 用户管理页面改进计划

## 📋 执行摘要

**目标**: 全面提升用户管理页面的功能性和用户体验
**预计工期**: 5 个阶段，可迭代执行
**优先级**: P0(高) > P1(中) > P2(低)

---

## 🎯 改进路线图

### Phase 1: 搜索与筛选功能 (P0 - 高优先级)
**目标**: 让用户能快速找到目标用户

#### 任务清单
- [ ] 1.1 添加用户搜索框（支持用户名、昵称模糊搜索）
- [ ] 1.2 添加角色筛选器（全部/家长/孩子/管理员）
- [ ] 1.3 添加性别筛选器（全部/男/女/未设置）
- [ ] 1.4 筛选状态与 URL 同步（便于分享和刷新保留）
- [ ] 1.5 后端 API 支持搜索和筛选参数

#### 验收标准
- 搜索响应时间 < 300ms
- 筛选器支持多选和清除
- URL 参数能正确反映当前筛选状态

---

### Phase 2: 批量操作功能 (P1 - 中优先级)
**目标**: 提升批量管理效率

#### 任务清单
- [ ] 2.1 添加行选择功能（复选框）
- [ ] 2.2 添加全选/取消全选功能
- [ ] 2.3 批量删除功能（带确认对话框）
- [ ] 2.4 批量修改角色功能
- [ ] 2.5 批量操作工具栏（固定在底部）

#### 验收标准
- 支持同时选择最多 50 条记录
- 批量操作有进度提示
- 操作失败时显示具体错误信息

---

### Phase 3: 用户体验优化 (P0 - 高优先级)
**目标**: 修复 Bug，提升交互体验

#### 任务清单
- [ ] 3.1 **修复性别显示 Bug**（gender 为 none 时显示"未设置"）
- [ ] 3.2 添加表单验证（必填项、格式校验）
- [ ] 3.3 添加加载状态（表格骨架屏）
- [ ] 3.4 优化空状态（添加插画和引导）
- [ ] 3.5 添加操作成功/失败提示
- [ ] 3.6 添加手动刷新按钮

#### 验收标准
- 表单验证实时反馈
- 加载状态平滑过渡
- 空状态有明确的引导操作

---

### Phase 4: 数据展示增强 (P1 - 中优先级)
**目标**: 展示更多有价值的信息

#### 任务清单
- [ ] 4.1 显示用户头像/emoji
- [ ] 4.2 显示积分信息（总积分/可用积分）
- [ ] 4.3 显示最后登录时间
- [ ] 4.4 显示用户状态（启用/禁用）
- [ ] 4.5 添加用户详情抽屉/弹窗
- [ ] 4.6 优化表格列宽和布局

#### 验收标准
- 新增信息不影响页面加载速度
- 表格支持横向滚动
- 响应式布局适配移动端

---

### Phase 5: 代码重构 (P2 - 低优先级)
**目标**: 提升代码质量和可维护性

#### 任务清单
- [ ] 5.1 提取用户管理相关 hooks（useUsers, useUserForm）
- [ ] 5.2 拆分表格列为独立组件
- [ ] 5.3 拆分 Modal 表单为独立组件
- [ ] 5.4 添加单元测试
- [ ] 5.5 优化 TypeScript 类型定义

#### 验收标准
- 代码覆盖率 > 70%
- 组件职责单一
- 类型定义完整

## 🔧 技术实现细节

### API 接口扩展

```typescript
// GET /api/user?search=keyword&role=parent&gender=boy&page=1&limit=10
interface GetUsersQuery {
  search?: string;      // 搜索关键词
  role?: string;        // 角色筛选
  gender?: string;      // 性别筛选
  page?: number;        // 页码
  limit?: number;       // 每页数量
}
```

### 组件接口设计

```typescript
// UserFilters 组件
interface UserFiltersProps {
  filters: {
    search: string;
    role: string;
    gender: string;
  };
  onFilterChange: (key: string, value: string) => void;
}

// UserToolbar 组件
interface UserToolbarProps {
  selectedRows: string[];
  onBatchDelete: () => void;
  onBatchUpdateRole: (role: string) => void;
  onRefresh: () => void;
}
```
